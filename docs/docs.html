Cumulus


Description

This is the base command for the Stratocumulus CLI


Options

Name, shorthand Description
--help, -h Get details on how to use cumulus and its accompanying commands
--version Get the version of Stratocumulus you have installed

Child Commands

Name Description
cumulus init Initialize a Stratocumulus Project
cumulus start Start the initialized Stratocumulus Project
cumulus stop Safely stops any running Stratocumulus project at current directory
cumulus kill Forcefully stops any running Stratocumulus project at current directory
cumulus shell Returns a shell within a container running the web application
cumulus restart Restarts a specifica service or entire project. This is the same as running cumulus stop and cumulus start
cumulus logs Specifies the logs of a certain service or all services in project

Cumulus Init


Description

Initialize all specified services


Usage

cumulus init [service] [OPTIONS]

Options

Name Default Description
--clean Clean the existing images..blah blah
--project-name=<project> Defaults to folder name This flag sets the project name in the configuration file to the one specified

Example


Command:
cumulus init django mysql

Output:
Creating network "webapp_default" with the default driver
Going to remove webapp_django_run_1
Removing webapp_django_run_1 ... done
Going to remove webapp_mysql_run_1
Removing webapp_mysql_run_1 ... done

If this command is successfull a docker-compose.yml file will be created, configuring the specified services.
Furthermore, any configurable service will have a folder created for it in the directory from which the command was run.
This includes services such as Django and MySql

Cumulus Start


Description

Start all the initialized services


Usage

cumulus start (<service>..)

Options

Name Description
--latest This option will pull the latest image from the Stratocumulus docker hub

Example


Command:
cumulus start

Output:
Creating webapp_mysql ... done
Creating webapp_django ... done
django app available at localhost:<port> logs available using `cumulus logs [<service>] [-f]`

If this command is successfull all initialized services will be running.
A port will be in the output specifying where a webapp is running.
This command can be run with or without explicit services in the command.
With no services specified all will be started, otherwise only those requested.
Note: <port> will change everytime depending on what ports are available on your machine.
