Installation

Download the latest release of Stratocumulus from here and follow the instructions

Requirements

Stratocumulus requires Docker to run, so please ensure you have it installed

Commands

Name Description
cumulus init Initialize a Stratocumulus project
cumulus start Start the initialized Stratocumulus project
cumulus stop Safely stops any running Stratocumulus project at current directory
cumulus kill Forcefully stops any running Stratocumulus project at current directory
cumulus shell Returns a shell within a container running the web application
cumulus restart Restarts a specifica service or entire project
cumulus logs Specifies the logs of a certain service or all services in project

Options

Name, shorthand Description
--help, -h Get details on how to use cumulus and its accompanying commands
--version Get the version of Stratocumulus you have installed

Cumulus Init


Initialize services


Usage

cumulus init [service] [OPTIONS]

Options

Name Default Description
service The services that will be initialized
--clean Do a clean initialization from scratch, directories will be replaced
--project-name=<project> Defaults to folder name Sets the project name in the configuration file

Example


Command:
cumulus init django mysql

Output:
Creating network "webapp_default" with the default driver
Going to remove webapp_django_run_1
Removing webapp_django_run_1 ... done
Going to remove webapp_mysql_run_1
Removing webapp_mysql_run_1 ... done

Creates a docker-compose.yml and any necessary web app directories

Cumulus Start


Start all initialized services


Usage

cumulus start [<service>..]

Options

Name Default Description
service Defaults to all Services to start
--latest This option will pull the latest image from the Stratocumulus docker hub

Example


Command:
cumulus start

Output:
Creating webapp_mysql ... done
Creating webapp_django ... done
django app available at localhost:32773 logs available using `cumulus logs [<service>] [-f]`

Specified services will be started

Cumulus Stop


Safely stop a service


Usage

cumulus stop

Example


Command:
cumulus stop

Output:
Stopping webapp_django ... done
Stopping webapp_mysql ... done

All services will be stopped

Cumulus Kill


Forcefully teardown a service


Usage

cumulus kill

Example


Command:
cumulus kill

Output:
Removing webapp_django ... done
Removing webapp_mysql ... done
Removing network webapp_default

All services will be killed, corresponding containers will be removed

Cumulus Restart


Restart a service


Usage

cumulus restart [<service>...]

Options

Name Default Description
service Defaults to all List of services that will be restarted

Example


Command:
cumulus restart

Output:
Restarting webapp_mysql ... done
Restarting webapp_django ... done

Specifed services will be restarted

Cumulus Shell


Get direct access to a service container


Usage

cumulus shell [<service>] [--shell=<shell>]

Options

Name Default Description
service Defaults to web app Service to shell into
--shell The shell specified in $SHELL environment variable The shell that will be used [bash, zsh, sh]

Example


Command:
cumulus shell --shell=bash

Output:
root@13ec7332f1ab:/cumulus/django#

You will now have a shell into your web app container.

Cumulus Logs


Display service logs


Usage

cumulus logs [<service>] [-f]

Options

Name Default Description
service Defaults to web app Service to show logs for
-f Tail the logs

Example


Command:
cumulus logs django -f

Output:
Attaching to src_django
